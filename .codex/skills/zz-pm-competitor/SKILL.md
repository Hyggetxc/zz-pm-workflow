# 竞品拆解与差异化

## 定位

本技能用于竞品分析，不负责正式 PRD 或优先级排序。

## 适用输入

适用输入：

- 竞品名单
- 竞品链接或截图
- 对标模块
- 自家产品定位

不适用输入：

- 只有一个竞品，且没有明确分析目的
- 纯公司研究，无产品对标目标
- 只想排需求优先级

## 默认产出

默认应输出能指导产品动作的竞品结论，而不是信息堆砌。

默认产出应包含：

1. 分析范围
2. 单竞品拆解
3. 横向对比矩阵
4. 可借鉴点
5. 差异化建议

## 边界

- 公司背景研究 -> `company-research`
- 优先级排序 -> `zz-pm-prioritization`
- 路线图 -> `zz-pm-roadmap`

## 默认触发场景

- “帮我做竞品分析”
- “看看这几个竞品”
- “和竞品比一下”
- “对标分析”
- “差异化怎么做”

## 执行流程

### 1. 确认分析目标

先明确：

- 为什么分析
- 给谁看
- 关注哪些模块
- 自家产品当前阶段

### 2. 做四维拆解

默认四维：

- 策略
- 功能
- 体验
- 增长

### 3. 横向对比

至少形成：

- 功能矩阵
- 策略矩阵
- 体验对比

### 4. 输出建议

至少区分：

- 可借鉴
- 不可照抄
- 差异化切入点

## 标准输出格式

```markdown
## 分析范围
- 竞品名单：
- 分析目标：
- 重点模块：

## 单竞品拆解
### 竞品：xxx
- 策略：
- 功能：
- 体验：
- 增长：

## 横向对比
| 维度 | 我们 | 竞品A | 竞品B |
|---|---|---|---|

## 结论
### 可借鉴
- ...

### 不可照抄
- ...

### 差异化建议
- ...
```

## 常见误用

- 只罗列功能，不做判断
- 没有“我们”的参照系
- 把竞品拆解直接等同于路线图结论
