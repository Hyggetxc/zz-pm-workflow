# 调研问卷设计

本技能用于用户调研问卷设计，不负责访谈纪要整理或纯数据分析。

适用输入：

- 调研目标
- 目标人群
- 决策场景
- 样本与渠道约束

不适用输入：

- 只有一个模糊的“想调研一下”
- 只需要分析已有问卷结果
- 只需要访谈提纲，不做问卷

## 工作目标

输出可直接发放或进一步落地的问卷方案。

默认产出应包含：

1. 调研目标拆解
2. 研究假设
3. 问卷题目
4. 偏差审查
5. 样本与发放建议

## 默认触发场景

- “帮我设计问卷”
- “做个用户调研”
- “这份问卷有没有问题”
- “设计 NPS / 满意度调查”
- “需求调研怎么出题”

## 边界

- 访谈与发现流程 -> `discovery-process`
- 优先级排序 -> `zz-pm-prioritization`
- 数据结果分析 -> `zz-pm-analytics`

优先使用本技能的判断标准：

- 用户明确要“问卷”
- 目标是“把研究目标变成可发放题目”
- 输出重点是“题目、偏差审查、样本与发放建议”

不应停留在本技能的场景：

- 用户要的是访谈提纲、深访流程或完整 discovery 方案 -> `discovery-process`
- 用户已经有问卷结果，只想解读结果 -> `zz-pm-analytics`

## 执行流程

### 1. 拆解目标

先明确：

1. 核心问题
2. 决策场景
3. 目标人群
4. 时间与样本约束

### 2. 生成研究假设

每个问题都要对应：

- 要验证什么
- 为什么验证
- 适合用什么题型

### 3. 设计题目

默认结构：

- 筛选题
- 预热题
- 核心题
- 背景题
- 开放题

### 4. 做偏差审查

检查：

- 诱导题
- 双重问题
- 跳转逻辑
- 选项是否 MECE
- 是否有不适用选项

### 5. 输出样本与发放建议

至少给出：

- 样本建议
- 渠道建议
- 发放时机
- 回收注意事项

## 标准输出格式

```markdown
## 调研背景
- 目标：
- 决策场景：
- 目标人群：

## 研究假设
| # | 子目标 | 假设 | 题型建议 |
|---|---|---|---|

## 问卷设计
### Q1
- 题型：
- 题目：
- 选项：
- 设计意图：

## 偏差审查
- ...

## 样本与发放建议
- ...
```

## 常见误用

- 没有假设就直接堆题目
- 题目太长、太多、太诱导
- 不给样本和发放建议
- 把问卷设计和访谈方案混为一体
