# 初评版 PRD 模板

说明：

- 本模板用于首次评审阶段。
- 它不是正式交付版 PRD 的简化复制，而是专门服务“原型完成后首次评审”的文档。
- 核心目标是让评审快速聚焦业务流程、涉及页面修改、关键逻辑交互和待确认项。
- 一些偏正式交付的信息可以不在本阶段展开，留到正式交付版 PRD 补全。

## 二、背景与目标

只写本轮评审真正相关的内容。

建议覆盖：

- 为什么要做这次改动
- 当前问题是什么
- 本次希望通过原型和评审先确认什么
- 本次评审的目标结果是什么

建议句式：

- 业务背景：
- 当前问题：
- 本次目标：
- 本次不讨论：

## 三、核心业务流程

这是初评版 PRD 的核心章节之一。

要求：

- 必须明确业务主流程
- 必须写清关键分支
- 必须标出本次评审重点关注的节点

建议写法：

### 3.1 主流程

按步骤描述：

1. 用户进入 `<页面 / 场景>`
2. 用户执行 `<关键动作>`
3. 系统出现 `<页面变化 / 状态变化>`
4. 用户进入 `<下一步>`
5. 最终达成 `<业务结果>`

### 3.2 关键分支

列出：

- 正常流程
- 异常流程
- 中断流程
- 回退流程

### 3.3 本次重点确认节点

只列真正需要评审确认的节点：

| 节点 | 当前方案 | 待确认点 |
|---|---|---|
| | | |

## 四、涉及页面与修改范围

这是初评版 PRD 的核心章节之二。

要求：

- 必须明确本次涉及哪些页面
- 必须明确每个页面是“新增 / 修改 / 删除 / 不变但受影响”
- 必须说明本次修改到什么范围

建议表头：

| 页面名称 | 页面类型 | 变更类型 | 本次修改范围 | 是否原型已覆盖 |
|---|---|---|---|---|
| | 列表页 / 表单页 / 详情页 / 专题页 / 弹窗 | 新增 / 修改 / 删除 / 受影响 | | 是 / 否 |

说明：

- 页面类型帮助快速建立评审上下文
- 修改范围不要写空话，要写模块级变化

## 五、页面详细说明

这是初评版 PRD 的主体内容。

每个页面建议按以下结构输出。

### 5.x 页面名称

#### 5.x.1 页面目标

说明该页面在整个业务流程中的作用。

#### 5.x.2 本次修改点

建议表头：

| 模块 | 当前情况 | 本次修改 | 修改目的 |
|---|---|---|---|
| | | | |

要求：

- 先写模块，不直接铺字段
- 只写本次评审相关变化
- 不把没有变化的模块写太重

#### 5.x.3 页面结构与交互

建议按“从上到下、从左到右”的顺序描述，但先模块后字段。

推荐句式：

- 模块名
  - 展示范围：
  - 展示顺序：
  - 字段说明：
  - 字段交互：

#### 5.x.4 关键逻辑

这是每页必须有的部分。

建议表头：

| 功能模块 | 元素 | 逻辑说明 |
|---|---|---|
| | | |

逻辑说明必须优先写：

- 入参校验
- 触发条件
- 页面变化
- 成功 / 失败反馈
- 与其他页面或状态的关系

#### 5.x.5 待确认项

这是初评版 PRD 和正式版 PRD 的关键差异点。

每个页面都建议单独列出待确认项，而不是全部堆到文档最后。

建议表头：

| 待确认项 | 当前方案 | 备选方案 / 风险 | 是否阻断进入正式版 |
|---|---|---|---|
| | | | 是 / 否 |

要求：

- 不要写泛泛的“待补充”
- 要写清到底在确认什么
- 如果有方案分歧，要写当前倾向方案

## 六、关键逻辑与交互

这是初评版 PRD 的核心章节之三。

它不重复逐页内容，而是集中写跨页面、跨模块的重要逻辑。

建议覆盖：

- 页面跳转关系
- 提交 / 保存 / 删除 / 确认逻辑
- 主按钮路径
- 弹窗触发条件
- 状态切换条件
- 列表与筛选的联动关系
- 页面 A 改动对页面 B 的影响

建议表头：

| 类型 | 场景 | 逻辑说明 | 涉及页面 |
|---|---|---|---|
| | | | |

## 七、页面状态说明

初评版 PRD 只保留和评审有直接关系的状态，不强求全状态穷举。

建议优先覆盖：

- 默认状态
- 空状态
- 成功状态
- 失败状态
- 权限受限状态
- 弹窗状态

建议表头：

| 页面 / 模块 | 状态 | 触发条件 | 页面表现 | 用户动作 |
|---|---|---|---|---|
| | | | | |

建议按优先级汇总：

| 优先级 | 待确认项 | 影响范围 | 建议决策时间 |
|---|---|---|---|
| P0 / P1 / P2 | | | |

## 九、初评结论与后续动作

建议明确：

1. 哪些方案已确认
2. 哪些问题待补充
3. 哪些页面需要继续改原型
4. 是否进入正式交付版 PRD
