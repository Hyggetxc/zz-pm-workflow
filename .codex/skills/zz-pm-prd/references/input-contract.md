# 输入目录约定

## 最小可用

- `screenshots/`：页面截图（按页面命名）
- `scope.md`：本次覆盖页面范围

## 推荐

- `examples/`：历史高质量文档（建议 3-10 篇）
- `rules.md`：硬规则（优先级最高）
- `glossary.md`：术语/缩略词定义
- `prototype/`：原型源码或页面链接（HTML / Pencil / 截图均可，需在 PRD 中写清来源）
- `flows/`：已有流程图素材（可选）

## rules.md 建议字段

### 1) 文档结构
- 是否完整 PRD
- 是否固定 `4.2 功能清单`
- 是否禁用总结章节
- 截图缺失时是阻断定稿还是允许草稿版输出

### 2) 表格结构
- 功能清单表头（建议：平台/一级功能/二级功能/优先级/说明）
- 功能详设表头（建议：功能模块/元素/实现归属/逻辑说明）
- 是否每个 `5.x` 需求单元单独表格或分段说明
- 列宽比例（如 18/26/12/44）

### 3) 功能清单颗粒度
- 是否只允许模块级能力进入 `4.2`
- 哪些动作必须并回说明列（如上架/下架/删除/导出）
- 哪些占位模块要压缩为汇总说明

### 4) 页面模板
- 列表页模块顺序
- 表单页模块顺序
- 详情页模块顺序
- 占位页是否允许独立成需求单元
- 官网 / Landing / SEO / 对比页的页面目标、阅读路径、CTA 与转化路径
- 业务后台 / 管理端的列表、筛选、表格、表单、权限、异常和状态机

### 5) 元素写法
- 字段句式：`字段名，字段类型，字段要求`
- 字段顺序：从上到下、从左到右
- 列表模块是否固定写：展示范围/展示顺序/字段说明/字段交互
- 通用元素（面包屑、标题、返回按钮）是否压缩描述

### 6) 逻辑写法
- 强制覆盖：前端元素、交互行为、后端可感知业务逻辑、入参校验、状态流转、计算口径、异常提示
- 每条功能详设必须标注实现归属：前端、后端、前后端、非代码项
- 字段规则是否要求阈值/并列/边界，并写入对应 `5.x` 需求单元
- 禁止纯技术实现细节
- 禁止“用于标识/用于识别”类空话

### 7) 字段确定性
- 枚举值是否必须展开
- 若枚举值不明，是否列为待补充项
- 搜索/联想/模糊匹配/精确匹配是否必须说明

### 8) 流程图策略
- 哪些模块必须补流程图
- 流程图必须放在对应 `5.x` 需求单元内
- Markdown 交付优先保留 Mermaid 源码；用户明确要求图片时再渲染为本地图片

### 9) 交付策略
- 输出目标（当前项目默认本地 Markdown `.md`）
- 是否明确禁止 `docx` 输出
- 更新方式（局部更新优先；仅用户明确要求时整文重写）
- 文件命名规则（默认 `docs/<版本号或日期> <需求名称>正式版 PRD.md`）

### 10) AI 实现依据
- 用户是否明确要求给技术或 AI 用于生成 / 修改代码
- 用户是否明确要求另行输出 `AI 实现依据` 独立文档
- 实现影响是否明确标注：前端 / 后端 / 前后端共同 / 非代码项
- 实现影响范围是否拆清：标准路径描述、影响角色、影响入口、影响状态、影响权限、影响已有业务逻辑、不影响范围
- 标准路径描述是否已写成业务路径；Pencil nodeID、Figma nodeID 或画布内部节点 ID 是否仅作为 `设计 / 原型补充引用`；新增页面 / 模块是否按 `本期新增` 标记
- 后端业务契约是否写清：触发条件、输入字段、输出结果、业务校验、状态变更、幂等 / 重复提交规则、权限判断、失败结果、是否影响历史数据
- 是否写清前端实现约束、后端业务契约、前后端共同规则、不做范围和 AI 校验点

## 默认交付行为

1. 当前项目默认只输出本地 Markdown（`.md`）。
2. 不生成 `docx`，也不再保留 `docx -> md` 的降级逻辑。
3. 默认不启用飞书交付流程。
4. PRD 交付不切换到 Word / `docx` 输出。

## 当前项目文档阶段建议

### 正式版 PRD

建议输入：

- `scope.md`
- `rules.md`
- `glossary.md`
- `screenshots/`
- `prototype/`
- `flows/`（如有）

特点：

- 优先附页面截图
- 强调结构完整、流程清晰、可交付
- 仅当用户明确要求 AI 生成、修改代码或实现影响拆解时，另行输出 `AI 实现依据`，并明确区分前端、后端、前后端共同和非代码项
- 默认输出 `.md`

## 待补充信息清单（建议输出）

以下信息缺失时，建议统一列入“待补充信息”：
1. 页面截图
2. 枚举值
3. 状态取值
4. 权限范围
5. 统计口径
6. 历史数据处理方式
7. AI 实现依据中的前端 / 后端归属
8. AI 实现依据中的标准路径描述
