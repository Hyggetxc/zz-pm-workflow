# 输入目录约定

## 最小可用

- `screenshots/`：页面截图，按页面命名
- `scope.md`：本次覆盖页面范围

## 推荐

- `examples/`：历史高质量文档，建议 3 到 10 篇
- `rules.md`：硬规则，优先级最高
- `glossary.md`：术语或缩略词定义
- `prototype/`：原型源码、页面链接或页面说明
- `flows/`：已有流程图素材，可选

## `rules.md` 建议字段

### 1. 文档结构

- 是否完整 PRD
- 是否固定 `4.2 功能清单`
- 是否保留或删除总结章节
- 是否需要单列“需求范围说明”
- 截图缺失时是阻断定稿还是允许草稿版输出

### 2. 表格结构

- 功能清单表头
- 功能详设表头
- 是否每页单独表格
- 列宽比例

### 3. 功能清单颗粒度

- 是否只允许模块级能力进入 `4.2`
- 哪些动作必须并回说明列
- 哪些占位模块要压缩为汇总说明

### 4. 页面模板

- 列表页模块顺序
- 表单页模块顺序
- 详情页模块顺序
- 占位页是否允许独立成章

### 5. 元素写法

- 字段句式：`字段名，字段类型，字段要求`
- 字段顺序：从上到下、从左到右
- 列表模块是否固定写：展示范围 / 展示顺序 / 字段说明 / 字段交互
- 通用元素是否压缩描述

### 6. 逻辑写法

- 强制覆盖：入参校验、状态流转、计算口径、异常提示
- 复杂字段是否要求阈值、并列、边界规则
- 禁止纯技术实现细节
- 禁止“用于标识 / 用于识别”类空话

### 7. 字段确定性

- 枚举值是否必须展开
- 若枚举值不明，是否列为待补充项
- 搜索、联想、模糊匹配、精确匹配是否必须说明

### 8. 流程图策略

- 哪些模块必须补流程图
- 流程图放置位置
- 渲染优先级，建议图片优先、文本兜底

### 9. 交付策略

- 输出目标，默认本地 `docx`
- 若启用在线文档，是否要求显式提供文档链接或 ID
- 若启用在线文档，配置缺失时是否阻断
- 更新方式，建议局部更新优先
- 权限要求

## 默认交付行为

1. 默认输出本地 `docx`。
2. 仅当 `docx` 生成能力不可用时，降级为本地 `md`，并返回降级说明。
3. 默认不启用任何特定在线文档交付流程。
4. 仅当用户明确要求某种在线文档平台时，才切换到对应交付模式。

## 文档阶段建议

### 阶段 A：初评版 PRD

建议输入：

- `scope.md`
- `rules.md`
- `glossary.md`
- 当前需求对话纪要或业务说明

特点：

- 可无截图
- 可无原型
- 允许存在待补充信息
- 目标是服务首次评审

### 阶段 B：正式交付版 PRD

建议输入：

- `scope.md`
- `rules.md`
- `glossary.md`
- `screenshots/`
- `prototype/`
- `flows/`，如有

特点：

- 优先附页面截图
- 强调结构完整、流程清晰、可交付

## 待补充信息清单

以下信息缺失时，建议统一列入“待补充信息”：

1. 页面截图
2. 枚举值
3. 状态取值
4. 权限范围
5. 统计口径
6. 历史数据处理方式
