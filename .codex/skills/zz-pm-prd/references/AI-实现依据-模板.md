# AI 实现依据模板

说明：

- 本模板用于技术实现、AI 生成代码或 AI 修改代码前的产品级实现依据。
- 本文档独立于正式版 PRD，不合并到 PRD 正文。
- 本文档不写数据库表结构、接口路径、服务拆分或代码方案。
- 本文档必须写清“改哪里、改什么、不改什么、如何验收”。
- 每条影响必须标明归属：`前端`、`后端`、`前后端共同`、`非代码项`。

路径填写规则：

- 必须提供标准路径描述，不允许只使用 Pencil nodeID、Figma nodeID 或画布内部节点 ID 作为页面定位。
- 使用业务可读路径，例如：`前台 / 个人中心 / 订单详情 / 续费配置卡片`。
- 本期新增页面写：`本期新增 / <页面名称>`。
- 本期新增模块写：`<已有页面路径> / 本期新增 / <模块名称>`。
- Pencil nodeID、Figma nodeID、画布内部节点 ID、画板名或截图名可以写入 `设计 / 原型补充引用`，用于追溯设计来源。
- 如果只知道原型画板名称，先转译为业务路径；nodeID 只能作为补充引用，不得替代 `标准路径描述` 字段。
