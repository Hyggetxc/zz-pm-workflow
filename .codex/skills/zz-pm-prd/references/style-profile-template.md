# 风格画像模板

用于快速抽取“用户文档风格”，保证生成结果可直接用。

## 1. 结构偏好

- 是否采用完整 PRD 结构（默认 1~7 章）：
- 功能清单是否固定在4.2：
- 5章是否按“每个需求单元一图一表 / 分段说明”组织：
- 是否禁用总结章节：

## 2. 表格偏好

- 功能清单表头：
- 功能详设表头（默认4列：功能模块/元素/实现归属/逻辑说明）：
- 是否强制每个需求单元独立表格：
- 列宽比例（如18/26/12/44）：

## 3. 元素表达偏好

- 基础句式（默认）：`字段名，字段类型，字段要求`
- 列表模块固定骨架（信息汇总/主按钮/列表）：
- 是否要求“展示范围 + 展示顺序 + 字段说明 + 字段交互”：
- 字段顺序是否强制“从上到下、从左到右”：

## 4. 逻辑表达偏好

- 重点覆盖：入参校验/状态流转/计算口径/异常处理
- 实现归属取值：前端/后端/前后端/非代码项
- 字段规则是否需要在对应 `5.x` 需求单元内覆盖阈值、并列、边界：
- 禁止的技术化描述：

## 5. 序号与排版偏好

- 一级序号：`1.2.3`
- 二级序号：`a.b.c`
- 同级要点是否允许空行：
- 信息架构是否使用代码块：

## 6. 术语偏好

- 关键术语：
- 禁用术语：
- 页面命名是否保留英文路径：

## 7. 交付与更新偏好

- 默认交付目标（默认本地 Markdown `.md`）：
- 默认更新策略（局部更新/整文重写）：
- 流程图形式（必须放在对应 `5.x` 需求单元内；Mermaid / 图片路径 / 文本兜底）：

## 常见禁用表达

- 待确认/待定/可选/视情况
- 研发自行决定
- 不可验收的泛化描述
