# 反例与修正

## 1. 功能清单反例

反例：
- 课程列表
- 课程上架
- 课程下架
- 删除课程

问题：
- 把列表内动作拆成了和页面并列的二级功能，导致 4.2 与 5 章重复。

修正：
- 在 4.2 只保留“课程列表”。
- 在说明列写清：列表内包含编辑、上架、下架、删除等操作。

## 2. 元素反例

反例：
- 页面标题，文本，固定为“课程管理”。
- 用于确认当前页面为课程主列表。

问题：
- 这是页面复述，不是业务定义。
- “用于确认/用于标识/用于展示”属于空话。

修正：
- 非关键通用元素可压缩或省略。
- 优先写对业务有影响的模块：筛选区、列表、状态、分页、提交按钮、关键弹窗。

## 3. 字段定义反例

反例：
- 学科，单选，取自学科字典。

问题：
- 没有给出研发可直接实现的枚举值。

修正：
- 若原型/代码中有明确值，直接展开：数学、语文、英语。
- 若原型/代码中没有明确值，不要假装写清楚，应列入“待补充信息”。

## 4. 截图反例

反例：
- 原型截图：本次未提供独立截图，页面以源码原型为准。

问题：
- 这会把正式 PRD 写成半成品。

修正：
- 能补截图就补截图。
- 不能补截图时，将该页面标记为“草稿版页面说明”，并把截图缺失列入待补充项。

## 5. 占位页反例

反例：
- 为每个占位导航页都单独写完整页面章节。

问题：
- 低价值内容占满第 5 章，冲淡主链路。

修正：
- 默认合并为“占位模块汇总说明”。
- 仅当用户明确要求逐页展开时，才给每个占位页独立章节。

## 6. 后端逻辑重复反例

反例：
- `5.3 规则认证与状态治理` 已经写了删除规则认证、流量耗尽认证拦截和心跳踢下线。
- `7.1 字段与枚举` 又写 `is_deleted = 0 才允许认证通过`。
- `7.2 异常处理矩阵` 又写已删除规则、流量耗尽、规则异常失效的认证或心跳结果。

问题：
- 第 `7` 章把第 `5` 章的后端业务逻辑重新归纳了一遍，测试和技术会看到重复口径。
- 后端规则被集中堆到通用章节，削弱了 `4.1 -> 4.2 -> 5` 的模块对应关系。

修正：
- 删除第 `7` 章中的重复规则。
- 将 `is_deleted`、流量耗尽、规则异常或失效、心跳踢下线等逻辑拆回第 `5` 章对应模块。
- 不再保留独立第 `7` 章；字段、异常、状态和流程图都放回对应 `5.x` 需求单元。
