# A/B 实验设计

**EN Summary:** Designs A/B experiments with hypotheses, variants, metrics, sample sizing, and decision rules.

## 定位

本技能用于实验方案设计，不负责纯数据分析、埋点文档或 PRD 正文。

## 适用输入

适用输入：

- 实验目标
- 可改动点
- 当前基线数据或流量规模
- 可接受实验周期
- 业务约束与风险边界

不适用输入：

- 只想分析已经完成的实验结果
- 只想设计埋点，不做实验方案
- 没有目标或没有可改动变量

## 默认产出

默认应输出一份能够直接用于评审和执行的实验方案。

默认产出应包含：

1. 实验假设
2. 分组与流量方案
3. 指标体系
4. 样本量与周期估算
5. 判定与止损规则

## 边界

`zz-pm-workflow` 负责：

- 判断当前是不是实验类任务
- 判断应先读取哪些项目文档
- 将任务路由到本技能

本技能负责：

- 假设设计
- 分组与流量设计
- 指标体系
- 样本量与周期估算
- 成功、失败、止损标准

`zz-pm-tracking-spec` 负责：

- 埋点与字段设计

`zz-pm-postmortem` 负责：

- 实验结束后的复盘

规则：

- 若用户要做实验设计、灰度方案、判定规则，使用本技能
- 若用户要补埋点细节，切到 `zz-pm-tracking-spec`
- 若用户要总结实验结果，切到 `zz-pm-postmortem`

## 默认触发场景

以下任务优先使用本技能：

- “帮我设计 A/B 实验”
- “这个需求要不要做实验”
- “写实验方案”
- “怎么做灰度验证”
- “帮我算样本量”
- “定义实验判定标准”

## 执行流程

### 1. 明确实验假设

假设至少包含：

1. 改动内容
2. 目标指标
3. 预期变化方向
4. 因果逻辑

优先使用以下模板：

- 如果我们做了什么改动
- 那么哪个指标会如何变化
- 因为背后的行为逻辑是什么

### 2. 设计实验单元与分组

至少明确：

1. 分流对象
2. 分组方式
3. 流量比例
4. 风险控制方式
5. 是否存在混杂变量

默认建议：

- 常规实验优先 `50/50`
- 高风险改动优先小流量起步
- 若无法用户级分流，再退到设备级或会话级

### 3. 设计指标体系

默认分三层：

- 核心指标
- 护栏指标
- 辅助指标

要求：

- 核心指标尽量只保留 1 个
- 护栏指标用于约束负向影响
- 辅助指标只用于解释，不用于最终判定

### 4. 估算样本量与周期

优先明确：

1. 基线值
2. 最小可检测效应
3. 显著性水平
4. 统计功效
5. 日均流量

若无法准确计算，至少给出：

- 估算所需参数
- 当前是否具备实验可行性
- 样本量不足时的替代建议

### 5. 设计判定与止损规则

至少明确：

1. 什么叫成功
2. 什么叫失败
3. 什么情况下继续观察
4. 什么情况下提前止损

止损规则至少考虑：

- 护栏指标恶化
- 系统性能或稳定性问题
- 实验分流异常

### 6. 做一致性检查

输出前至少检查：

- 假设是否可证伪
- 是否同时改了过多变量
- 指标是否能被当前埋点支持
- 周期是否覆盖完整观察窗口
- 判定标准是否明确且可执行

## 输出要求

要求：

- 假设必须可量化、可证伪
- 指标必须定义清楚，不写空话
- 不能忽略护栏指标
- 若数据不足，明确标记估算和假设
- 若当前不适合做实验，要直接说明原因

## 常见误用

避免以下情况：

- 把“想试试”当成实验假设
- 同时改太多变量，导致无法归因
- 只有核心指标，没有护栏指标
- 样本量明显不够，却仍然给确定性结论
- 用户其实要做实验复盘，却只输出实验设计
