# URL 到本地原型工程

## 定位

本技能用于把网页或站点参考复刻为本地原型工程，优先工程化交付。

## 边界

- 截图复刻 -> `zz-pm-proto-image`
- `.pen` 设计稿 -> `zz-pm-proto-pencil`
- 只做原型路径判断 -> `zz-pm-prototype-workflow`

优先使用本技能的判断标准：

- 主要输入是 URL 或在线页面
- 目标是交付本地可运行的原型工程
- 需要保留后续扩展、迭代和代码级改造空间

不应停留在本技能的场景：

- 用户只有截图，且只要快速页面复刻 -> `zz-pm-proto-image`
- 用户明确要求 `.pen` 设计稿 -> `zz-pm-proto-pencil`
- 用户只想判断原型形式，还没决定做哪种交付 -> `zz-pm-prototype-workflow`

若同时给了 URL 和截图：

- 结构以 URL 为主
- 视觉细节以截图为主
- 交付仍以本地工程为准

## 适用输入

适用输入：

- 参考 URL
- 在线页面
- 补充截图
- 工程化交付要求

不适用输入：

- 只有截图，且只要轻量页面复刻
- 明确要求 `.pen` / Pencil 设计稿
- 只想判断原型路线，不做具体执行

## 默认产出

默认应包含：

1. 本地可运行的原型工程
2. 主要页面结构与样式还原
3. 运行说明与可修改点

## 默认触发场景

- “克隆这个网站做原型”
- “根据这个 URL 做一个本地化页面”
- “做成可运行工程”

## 执行流程

1. 读取 URL 结构与页面特征
2. 结合截图或视觉参考判断设计语言
3. 输出本地原型工程
4. 补运行说明和可修改点

## 常见误用

- 用户只要静态原型，却强行上工程化
- 只还原视觉，不交付本地运行结构
- 有 URL 却完全不利用页面结构信息
